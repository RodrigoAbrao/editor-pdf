# Contribuindo

## Padrões
- Documentação em português
- Decisões importantes devem virar um ADR em `docs/DECISIONS/`

## Propostas de mudança
- Abra uma issue descrevendo: problema, proposta, trade-offs, impacto.

## Checklist (quando houver código)
- Testes passando
- Lint/format ok
- Endpoints documentados (docs/API.md)
